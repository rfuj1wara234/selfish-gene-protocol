# Selfish Gene Protocol

> A protocol for human × AI agent collaboration.
> Inspired by Richard Dawkins' *The Selfish Gene*.

---

## What is this?

**A protocol is an innovation gene — designed to self-replicate.**

Just as DNA propagates itself through organisms, this protocol propagates through communities. The company or organization that hosts it is merely a vessel. The protocol is the thing that matters.

The goal is not to build a great company. The goal is to define a protocol so valuable that it outlives any single organization — like Gaudí's Sagrada Família, designed to be completed long after its creator is gone.

---

## The Problem

Most thinking about human × AI collaboration stops at Layer ①.

```
① Individual + AI Agent
   One person uses AI to do the work of many.
   → Most of the conversation today stops here.

② Multiple humans + AI Agents collaborating
   A team where humans and AI agents work together,
   governed by a shared protocol.
   → Very few people have thought this through.

③ An ecosystem of ② instances
   Many ②-type communities interacting,
   sharing learnings, forking each other's protocols.
   → This is the endgame.
```

This protocol is designed for Layer ②, with Layer ③ as the destination.

## Core Philosophy

| Biology | This Protocol |
|---|---|
| Gene (DNA) | Protocol |
| Organism | Company / Community |
| Natural selection | Market competition |
| Gene propagation | Protocol replication |

**The organization is the vessel. The protocol is what survives.**

---

## The Portfolio Principle

- Many communities run the protocol with different parameters
- Some fail, some succeed — natural selection in action
- Accumulated learnings raise the **baseline** for everyone who comes after
- This is a **portfolio with compounding returns**, not a single bet

---

## The Contribution Rule

**You must give to receive.**

Use it freely. But contribute your implementation log — what you tried, what worked, what failed. Communities that consume without contributing lose access to the shared knowledge base.

The accumulated implementation logs *are* the protocol's primary asset.
